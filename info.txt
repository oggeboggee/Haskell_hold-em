




The model:

We have three layers.

Layer 1: Clients (WS.connections)
Just connections that can come and go.

Layer 2: Lobby waiting room
"Registered players", all are here by default

Layer 3: Engine table seats
Players "locked in" to the hand at the start of a hand.


We want: 

JoinMsg -> always add to the Lobby
Then table seating happens ONLY in startNewHand.



Or: 

ClientMsg ActionMsg
-> Resolve players
-> check they are seated
-> run applyEvent on activeGame.table
-> update Active.game


The table should decide what state it is in. Server reacts.

We have a applyEvent which applies events in the engine.

Server that receives ActionMsg


We want to achieve: The server decide when the engine advances but the engine decided what advancing means.
Server becomes the dictator and engine the executor.

Server responsibility:
Receive ClientMsg
call applyEvent
broadcast GameEvents
ask enggine, what next?
